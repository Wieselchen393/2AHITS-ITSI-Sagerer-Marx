## gdb
### Befehle:
- b main              # set a breakpoint at function main
- run                 # start the program from the beginning
- disas               # disassemble the current function
- disas main          # disassemble the function main

- x/i 0xabcd          # show the instruction at address 0xabcd
- x/5i 0xabcd         # show 5 instructions starting at address 0xabcd
- x/16xb 0x401000     # show 16 bytes of data at address 0x401000 in hex

- c                # continue program execution
- s                # execute the next instruction and step into functions
- n                # execute the next line but step over function calls
- si               # step into assembler instruction
- ni               # step over assembler instruction

# set memory
set {unsigned char}0x401000 = 0x90


## manipulation von pwcheck
- gdb starten mit dem Programmname als Argument
- break point bei main setzen mit $b main
- Programm mit $run ausführen
- mit $ disas das Programm durchlaufen lassen und dann die zu manipulierende Stelle lokalisiern
- debugger neustarten und die ersten 2 Befehle erneut eingeben
- $x/i $pc eingeben und dann mit ENTER oder ni zur gewünschten Stelle gehen
- Speicher adresse der zu manipulierende Instruction kopieren
  BSP:
```sh
0x000055ded88551c0 <+80>:    call   0x55ded88552a0 <_Z14check_passwordPKcS0_>
0x000055ded88551c5 <+85>:    cmp    $0x0,%eax
0x000055ded88551c8 <+88>:    je     0x55ded8855263 <main()+243>
0x000055ded88551ce <+94>:    lea    0xe55(%rip),%rdi        # 0x55ded885602a
0x000055ded88551d5 <+101>:   mov    $0x0,%al

```
- dann mit set {unsigned char}0x000055ded88551c8 = 0x90 , die Adresse muss die selbe sein wie der Instruction
- nop hat nur einen Byte, das heißt man muss alle Adresse bis zur nächsten Instruction durch nop ersetzen
```sh
(gdb) set {unsigned char}0x000055ded88551c8 = 0x90
(gdb) set {unsigned char}0x000055ded88551c9 = 0x90
(gdb) set {unsigned char}0x000055ded88551ca = 0x90
(gdb) set {unsigned char}0x000055ded88551cb = 0x90
(gdb) set {unsigned char}0x000055ded88551cc = 0x90
(gdb) set {unsigned char}0x000055ded88551cd = 0x90
```
- das Programm dann mit c fortsetzten und testen

- mit x/3i (RAM-Adresse) kann man darauf folgende Befehle anzeigen um die Richtigkeit der Stelle zu überprüfen
- die 3 gibt an wie viele Instruction nach der Angegebenen Adresse angezeigt werden
je = 6 Byte
